## 问题一：刷新时状态栏文字闪烁

### 现象
每隔几十秒余额刷新一次，期间状态栏文字会短暂变成 `"..."` 并闪烁。

### 根因
`RefreshBalance()` 开头调用了 `SetState("...", _yellow)`，在网络请求返回前将显示文字重置为占位符，造成视觉抖动。

### 修复
删除刷新开始时的占位符重置，余额静默在后台更新，拿到结果后直接刷新显示文字。控件同时启用了双缓冲（`OptimizedDoubleBuffer`）消除重绘闪烁：

```csharp
SetStyle(ControlStyles.AllPaintingInWmPaint
       | ControlStyles.UserPaint
       | ControlStyles.OptimizedDoubleBuffer
       | ControlStyles.DoubleBuffer
       | ControlStyles.ResizeRedraw, true);
```

## 问题三：任务栏控件背景 / 白色色块

### 现象
去掉背景色后出现白色色块；恢复背景色后又出现间歇性白色边框。

### 根因分析
- 移除 `Label` 子控件、改用自绘后，`BackColor = Transparent` 导致 WinForms 向父窗口（任务栏）请求背景，Explorer 绘制白底。  
- 圆角方案使用 `Region = new Region(path)` 裁剪，任务栏宿主会在 Region 外侧重绘背景，产生白色锯齿边。

### 修复方案：完全自绘 + 不使用 Region

```csharp
// OnPaintBackground：先填充底色再画圆角路径，不依赖 Region 裁剪
protected override void OnPaintBackground(PaintEventArgs e)
{
    e.Graphics.Clear(_panel);                          // 整个区域填充底色
    e.Graphics.SmoothingMode = SmoothingMode.AntiAlias;
    using GraphicsPath path = CreateRoundedPath(..., 6);
    using SolidBrush brush  = new SolidBrush(_panel);
    using Pen pen            = new Pen(_panelBorder, 1f);
    e.Graphics.FillPath(brush, path);
    e.Graphics.DrawPath(pen, path);                    // 只画路径，不裁剪 Region
}

// OnResize：不再设置 Region，只触发重绘
protected override void OnResize(EventArgs e)
{
    base.OnResize(e);
    Invalidate();
}
```

**关键原则**：`Region` 会告诉 Windows 把 Region 外的像素交给父窗口绘制，导致外部宿主画白底。改用路径填充后，窗口始终自己负责所有像素。

## 问题四：弹窗内容被截断（主要问题）

### 现象演变

| 版本 | 可见内容 |
|------|----------|
| 早期 | 无标题栏，只有 "¥ 42.51 / ● 可用"，子卡片和按钮不见 |
| 中期 | 标题栏可见，"¥ 42.51" 底部被截，子卡片和按钮不见 |
| 最终 | 全部内容正常显示 ✓ |

### 根因一：弹窗定位坐标系混用（高 DPI 下）

`_deskBandControl.RectangleToScreen()` 返回的是 **Win32 物理像素坐标**，而旧代码直接赋给 `Form.Location`（WinForms 逻辑坐标）。在 125% / 150% DPI 下，两套坐标相差 1.25–1.5 倍，导致弹窗大部分被推到屏幕底部以下，只有顶部一小段可见。

**修复：全程使用 Win32 物理坐标**

```csharp
// 先 Show() 让窗口句柄就绪，再用 Win32 定位
_flyout.Show();

Win32Pos.GetWindowRect(_flyout.Handle, out Win32Pos.RECT flyRect);
int physW = flyRect.Right - flyRect.Left;   // 物理宽高
int physH = flyRect.Bottom - flyRect.Top;

int screenW = Win32Pos.GetSystemMetrics(0); // SM_CXSCREEN（物理）
int screenH = Win32Pos.GetSystemMetrics(1); // SM_CYSCREEN（物理）

int fx = Math.Max(0, Math.Min(bandRect.Left, screenW - physW));
int fy = bandRect.Top - physH;
if (fy < 0)            fy = bandRect.Bottom; // 任务栏在顶部时改为向下弹出
if (fy + physH > screenH) fy = screenH - physH;

Win32Pos.MoveWindow(_flyout.Handle, fx, fy, physW, physH, true);
```

> **核心原则**：`RectangleToScreen` 与 `MoveWindow/GetWindowRect` 都在物理坐标系下工作，不要与 `Form.Location`（逻辑坐标）混用。

### 根因二：WinForms AutoScaleMode 导致窗口尺寸不确定

WinForms 的 `AutoScaleMode.Font`（默认值）会在窗体加载时根据 DPI/字体比例自动缩放控件尺寸，可能使 `this.Size` 与代码设定值不一致。

**修复：在构造函数中关闭自动缩放**

```csharp
this.AutoScaleMode = AutoScaleMode.None;   // 禁止 WinForms 自动缩放
this.MinimumSize   = new Size(W, H);       // 防止外部强制缩小
this.Size          = new Size(W, H);
```

### 根因三：20pt 字体在 150% DPI 下高度 = Label 高度，导致文字底部被截

20pt 字体在 150% DPI（144 dpi）下渲染高度 = `20 × 144 / 72 = 40px`，恰好等于 Label 原设定高度 40px，字体 descender 超出边界被裁掉。

**修复：增大 Label 高度留出余量**

```csharp
// 高度从 40 改为 52，在 150% DPI 下仍有 12px 缓冲
var val = new Label { ..., Size = new Size(W - 48, 52) };
```

```powershell
$p  = 'd:\项目代码库\02自动化-半成品\DeepSeek-DeskBand-main'
$vd = "$p\bin\Live-$(Get-Date -Format 'yyyyMMddHHmmss')\net48"
New-Item -ItemType Directory -Path $vd -Force | Out-Null
Copy-Item "$p\bin\Release\net48\*" $vd -Recurse -Force

$dll = "$vd\DeepSeekDeskBand.dll"
$ra  = 'C:\Windows\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe'

Start-Process $ra -ArgumentList ('"' + $old + '" /unregister /silent') -Verb RunAs -Wait
Start-Process $ra -ArgumentList ('"' + $dll + '" /codebase')           -Verb RunAs -Wait
```

## 关键经验总结

| 场景 | 教训 |
|------|------|
| 任务栏圆角控件 | 不要用 `Region` 裁剪，改用路径填充，自己绘制所有像素 |
| COM DeskBand 弹窗定位 | `RectangleToScreen` 返回物理坐标，必须配合 `MoveWindow` 使用，不能赋给 `Form.Location` |
| WinForms 窗体尺寸 | 代码创建的 Form 一定要设 `AutoScaleMode = None`，否则 DPI 变化时尺寸不可预期 |
| 高 DPI 字体 | Label 高度 = 字体点数 × DPI / 72，150% DPI 下 20pt = 40px，需要留至少 12px 缓冲 |
| DLL 热更新 | Explorer 锁定运行中的 DLL，必须输出到新时间戳目录再切换注册 |

### 需求
1. 余额低于 ¥10 时，任务栏控件背景变红，起到视觉预警作用。  
2. 弹窗底部增加"官网充值"按钮，点击跳转充值页面。

### 实现：低余额警告（DeskBandControl.cs）

新增可变背景色字段 `_bgColor`，刷新余额后解析数值，按阈值切换背景：

```csharp
private readonly Color _redBg = Color.FromArgb(180, 40, 40);
private Color _bgColor;   // 初始化为 _panel

// RefreshBalance() 中：
bool lowBal = decimal.TryParse(_currentBalance.TotalBalance,
    NumberStyles.Any, CultureInfo.InvariantCulture, out decimal balVal)
    && balVal < 10m;
_bgColor = lowBal ? _redBg : _panel;
SetState($"¥ {_currentBalance.TotalBalance ?? "—"}", lowBal ? Color.White : _green);

// OnPaintBackground：用 _bgColor 代替硬编码 _panel
e.Graphics.Clear(_bgColor);
using SolidBrush brush = new SolidBrush(_bgColor);
```
